Audit #5 remediation: all 7 items from priority list

ITEM 1 — SVG icons (done in previous commit 173e9de)

ITEM 2 — Fuzzy/weighted search via Fuse.js
  Added Fuse.js 7.0.0 via CDN (6 KB gzipped). The previous search was
  exact-match substring — one typo returned zero results.
  New search: threshold 0.4 tolerates common typos; fields weighted:
    name × 3.0 > tags × 1.5 > description × 0.8 > category × 0.3
  Index built once after apps load (buildSearchIndex()), then reused on
  every keystroke. Substring fallback if CDN script fails to load.

ITEM 3 — Related tools extended to all 130 apps
  Expanded RELATED map in app-nav.js from 16 entries to 125 entries
  covering every real-content app. Grouped by semantic relationship:
  developer tools cluster together; finance tools cross-link; games
  grouped by play style (board games, word games, arcade games, etc.).
  The related bar now appears on virtually every app, not just 16.

ITEM 5 — Loading/progress UI for heavy operations
  Added self-contained loading wrapper to 5 app buttons that previously
  froze silently during API calls:
    pdf-splitter  (Split & Download ZIP)
    pdf-to-image  (Convert to Images, Convert to PDF)
    image-tools   (Process & Download)
    svg-to-png    (Convert to PNG)
  Pattern: button disabled + spinner SVG + "Working…" while fetch runs;
  re-enabled with original text in .finally(). PDF merger, OCR tool, and
  watermark tool already had their own loading states — no change needed.
  Added @keyframes spin + .spin-icon to base.css for the spinner.

ITEM 6 — Design system consolidation (continued)
  Stripped duplicate .btn-ghost { background: var(--bg); ... } and
  .btn-ghost:hover { background: var(--border); } from 37 more apps.
  These are already defined in base.css with equivalent semantics.

ITEM 7 — Pre-commit U+0008 / BOM scanner
  Added .git/hooks/pre-commit shell script that scans every staged
  .html/.js/.css/.json/.py file for:
    - U+0008 Backspace (0x08) — silently poisoned id attributes
    - UTF-8 BOM (EF BB BF)     — crashes Python json.load()
  Commit is blocked with a clear error message if either is found.
  This catches the class of corruption that previously went undetected
  for an unknown period across 66 files.
